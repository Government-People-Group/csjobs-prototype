Accessibility Statement for the GOV.UK One Login Prototype

The prototype is intended for all people in central government service teams who want to explore journeys within the GOV.UK One Login service, so you understand where and how it will form part of your user journey.

GOV.UK One Login is committed to making this website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

This accessibility statement applies to the Prototype website and the prototype One Login pages it hosts.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliance(s) listed below.

Non-accessible content

The homepage and navigation page that allow users to jump to a specific page or view common user journeys in the prototype have been fully audited against WCAG 2.2 and should provide an accessible route into the prototype for users.

However, the prototype attempts to reflect the user journeys of the One Login service (currently in public beta) using the components available in the GOV.UK Prototype kit. In some cases where the designs do not match the components available in the GDS Design System or prototype kit, custom code is used. As a result, the source code and interactions created will not match the live, production-quality code, or go through the same process of testing and validation as code released to the live service.

Content entered into forms may not be validated in the same way as production code. In some cases the prototype may respond differently to user input compared to the live service, as it does not have access to the same third-party data sources or use the same hosting methods.

Please contact the team using the details below in the feedback and contact information section if you have questions about the accessibility of any part of the live service.

The initial version of the prototype published in January 2024 has been tested to ensure that the journey can be completed with keyboard only on a desktop and mobile device, and that all content is rendered correctly by a screen reader.

Any additions after that date may not have been tested. The prototype code will be constantly updated to match the live service.

Feedback and contact information

If you discover an accessibility issue with the current prototype which stops you completing a journey, please contact the One Login accessibility team at {{ accessibilityTeamEmail }} or for GDS staff on Slack at #di-accessibility-ask slack channel.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’).

If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).
